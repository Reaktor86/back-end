	1. Как устроен Битрикс, теория

Есть публичная часть, которая видна обычным пользователям и административная часть, доступ к которой может получить только контент-менеджер

База данных - это универсальное место хранения инфы на сайте со специальными программами, которые этой инфой могут управлять. Хранит динамическую инфу

Статическая инфа - это отдельные файлы (видео, картинки, текст, страница сайта).

Папка - это раздел сайта

Программный код хранится в папке bitrix и его нельзя изменять.

Файлы и папки проекта расположены здесь: Контент > Структура сайта

Инфоблоки – это пакеты однородной информации. Инфа может быть любой

Динамическая инфа может формироваться только инфоблоками

Модули - это программы, каждая из которых - это определённый функционал для сайта

С помощью компонентов модуль выводит инфу для просмотра

У каждого модуля свой набор компонентов

Когда создается файл, создается и страница

Если на странице не размещены компоненты, то обращений к модулям и базе данных не происходит

	2. Управление структурой

Логическая структура - расположение по смыслу
Физическая - распоожение файлов и папок на сервере

Инструмент Структура обеспечивает доступ к файлам и папкам прямо из публичной части.

Менеджер файлов позволяет выполнять разные операции с файлами (доступен из публич. и админ. частей)

Новые страницы и разделы на сайте можно создать с помощью Мастера создания страниц/разделов
Создать страницу -> Создать страницу

При создании раздела автоматически создается страница index.php
Создать раздел -> По шаблону -> Стандартная страница

Если собираюсь создать меню сайта, которое будет редко меняться, то лучше создавать статическое. Если нужно, чтобы при изменениях в меню автоматически подстраивался дизайн, то создавать динамическое.

Статическое меню подключает файлы и папки физической структуры сайта, динамическое меню подключает разделы и элементы инфоблока.

Средства навигации: меню, карта сайта, навигационная цепочка(хлебные крошки)

	3. Компоненты: что это и как их использовать

Компонент - это отдельный функционал на сайте, например, корзина, новости, faq и тд

Компонент можно добавить из публичной части - меню "редактирование страницы", справа

Редактировать размещенный компонет - в меню "редактирование страницы", непосредственно оттуда, где он размещен

Шаблон компонента можно редактировать, копировать, отключить


	4. Инфоблоки

Инфоблок предназначен для управления однородной по содержанию инфорамцией

Каждый инфоблок - это таблица с данными, например: название новости, картинка к новости, текст новости и т д.

Чтобы инфоблок отобразился на сайте, нужно сначала создать компонент

Элемент - это информация, которую мы размещаем в инфоблоках (товары, фотографии, статьи, справочники и т.д.).

Раздел - это логическая единица для группировки элементов.

Добавить тип инфоблоков:
Контент - Инфоблоки - Типы инфоблоков - Добавить новый тип

Добавить новый инфоблок:
Контент - Инфоблоки - Типы инфоблоков - <Мой тип> - Добавить инфоблок
(там же можно задать свойства, SEO, редактировать)

Свойства создаются отдельно для каждого инфоблока.
Значения свойства задаются в элементах инфоблока.
Показ значений свойства на сайте определяется настройками компонента.

Создать новый раздел инфоблока:
Контент - Каталоги

Нельзя создать раздел, если в типе инфоблока не поставлена галочка "использовать древовидную структуру"

Упорядочить список разделов можно с помощью индекса сортировки

Элементы разделов можно добавлять, изменять, удалять одноименными кнопками (вместо слова элемент может быть новость, товар и т.п.).

Данные инфоблоков можно импортировать или экспортировать в форматах CSV, XML, RSS

Чтобы вывести на сайт информацию из инфоблока, следует:
а) выбрать и разместить компонент на странице;
б) в настройках компонента указать текущий инфоблок.

КОНСПЕКТ ОСНОВНОЙ

$arParams - массив с входными параметрами компонента
$arResult - массив с результатами работы компонента для передачи в шаблон

внедрение компонента в index.php:
global $APPLICATION;
$APPLICATION->IncludeComponent(
    "web:simple", // неймспейс и название компонента
    "temp", // название шаблона компонента
    [],
    false
);

папка local: здесь можно размещать собственные шаблоны компонентов и сайтов в папках components и templates соответственно

при одинаковых названиях:
высший приоритет у local/templates/мой сайт/components
средний у local/components
низкий у bitrix/components
